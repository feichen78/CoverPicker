## 🚀 下一版本：v1.1.0 - Zoom 精修 + 状态持久化（预计 12-16 小时）

### 目标
实现 CoverPicker 的核心差异化能力——Zoom 精修，并解决状态丢失问题。

### 任务分解

#### 1. 状态持久化（SQLite）（约 4 小时）
- 设计数据库表结构（`video_state`）
- 实现 `StateManager` 类（保存/读取锁定、收藏、导出、已浏览）
- 在 `load_video`、`lock_selected`、`favorite_selected`、`export_selected` 中集成
- 测试跨会话恢复

#### 2. Zoom 精修 L1-L2 基础版（约 8 小时）
- 设计 Zoom 弹窗交互（双击截图后，弹出 Zoom 窗口）
- L1（±2s，9张）：在当前时间点周围 ±2s 范围内精细采样
- L2（±8s，12张）：在当前时间点周围 ±8s 范围，采样当前 + 相邻分区
- 支持在 Zoom 窗口中锁定/收藏/导出
- 支持从 Zoom 窗口返回 Grid

#### 3. Zoom 精修 L3-L4 扩展（约 4 小时）
- L3：跨全部分区拓展采样（约 16 张）
- L4：全局无差别全新重采样（约 25 张）
- 添加层级切换按钮和视觉指示

### v1.1.0 交付物
- ✅ 状态永久保存（重启不丢失）
- ✅ Zoom 精修 L1-L4 完整功能
- ✅ 可安装的 Python 包（`pip install` 即可运行）

## 🔧 后续版本：v1.2.0 - 导出增强 + 自定义分区（预计 10-14 小时）

### 目标
完善导出功能，支持用户自定义分区方式。

### 任务分解

#### 1. 批量导出收藏截图（约 3 小时）
- 添加"批量导出全部收藏"按钮
- 按视频名称创建子目录
- 导出后保持导出标记

#### 2. 导出无损视频片段（约 4 小时）
- 实现指定时间区间的 MP4 片段导出
- 使用 FFmpeg stream copy（无重编码）
- 在 Zoom 窗口或 Grid 中设置起止点
- 自动命名防覆盖

#### 3. 自定义分区功能（约 5 小时）
- 添加分区数量设置（例如 3~10 个分区）
- 或拖拽调整分区边界（交互更复杂，可能需更多时间）
- 自定义分区模板保存

#### 4. 用户自定义视频目录（约 2 小时）
- 添加"选择目录"按钮
- 保存用户偏好到配置文件

### v1.2.0 交付物
- ✅ 批量导出收藏截图
- ✅ 导出无损视频片段
- ✅ 自定义分区
- ✅ 自定义视频目录

## 🎯 生产版本：v2.0.0 - 独立可执行文件（预计 8-12 小时）

### 目标
将应用打包为独立可执行文件，用户无需安装 Python 环境即可运行。

### 任务分解

#### 1. 打包与分发（约 6 小时）
- 使用 PyInstaller 或 Nuitka 打包 Windows 可执行文件（.exe）
- 测试 macOS 打包（.app）可选
- 配置 FFmpeg 嵌入或提示用户安装
- 创建安装程序（Inno Setup 或 NSIS）

#### 2. 性能优化与稳定性（约 4 小时）
- 大文件超时优化（动态超时）
- 内存泄漏检查（长期运行）
- 缓存自动清理（按大小/时间）
- 错误上报机制（日志文件）

#### 3. 用户文档（约 2 小时）
- 用户手册（README.md / 帮助文档）
- 快速入门指南

### v2.0.0 交付物

## ⚠️ 风险与建议

| 风险              | 影响                            | 缓解措施                                  |
| ----------------- | ------------------------------- | ----------------------------------------- |
| Zoom 精修交互复杂 | 开发时间可能超出预估            | 先实现 L1-L2，L3-L4 可简化                |
| FFmpeg 打包体积大 | 安装包过大                      | 提示用户单独安装 FFmpeg                   |
