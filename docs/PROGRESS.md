# 📋 CoverPicker 项目进度日志

> ## 🚨 核心开发规则（必须遵守）
>
> 1. **提供完整代码**：任何时候，需要提供修改好的完整代码（可以适当精简），以便用户直接覆盖。**不允许仅给出片段代码、部分方法，让用户自行替换**。
> 2. **确认现象与需求**：除非用户仅给出报错日志，其他时候必须向用户确认错误或问题的现象，以及用户的需求，用户同意后再开始修改代码。
> 3. **统一提供完整文件**：任何时候，不需要告诉用户"我只修改了xxx，但是为了完整，我还是提供xx"。提供代码时统一遵照第1条处理。
> 4. **禁止擅自删改功能**：任何时候，不允许未经用户授权，删除或修改coverpicker已有的功能、布局、按钮等。如果确实需要更改，必须取得用户确认。如果不确定用户原来的代码都有什么功能，绝不允许擅自开始修改或生成简化的文件代码，务必向用户询问、索取原始代码。
> 5. **自查后输出**：任何时候不需要请用户确认修改方案是否正确。排除故障时，如果有必要，则在代码中增加调试日志，不需要请用户确认。给出修改代码之前，必须自查至少2次，确保无误才能输出代码。不允许修改一点就让用户测试，再反复修改。

## ✅ 已完成功能（v2.4.3）

### 核心交互
- 主界面布局（左-中-右），视频列表、分区按钮、截图网格、底部操作栏。
- 单击选中/取消选中，双击放大预览（滚轮缩放 + 拖动平移）。
- 全选/取消全选、键盘快捷键（Ctrl+A, Ctrl+D, Delete, Space, 方向键）。
- 分区切换、密度切换（9/12/16/25）。
- 排除区间设置（支持编辑已有区间、持久化、跨设备同步）。
- 收藏、锁定、导出、刷新未锁定、全部重抽。

### 后台与性能
- 异步 FFmpeg 帧提取（asyncio + 信号量控制并发）。
- 缓存复用（1 秒内时间差匹配）。
- 固定缓存目录（%TEMP%/CoverPicker_cache），退出不删除。
- 缓存统计（大小、文件数）及手动清理。
- 任务取消即时响应（切换分区时 kill 子进程）——**已实现**。
- 视频加载时界面立即更新（显示占位图、视频名称提前显示）。

### 数据持久化
- SQLite 数据库存储视频、分区、收藏、排除区间。
- 跨设备状态同步（通过 file_id 识别）。
- 自动备份（关闭窗口时备份至指定目录）。
- 崩溃报告生成（异常捕获 + 日志轮转）。

### 辅助功能
- 预览窗口（视频帧预览、时间轴选段、自定义分区、片段导出）。
- 智能监控（目录变化自动导入、定时扫描、删除检测；跳过_covers目录）。
- 导入视频（文件/文件夹，基于文件名去重）。
- 批量删除视频（仅删除数据库记录，不删除视频文件）。

### 收藏管理增强
- 查看收藏窗口按分区分组显示。
- 自适应列数（1~6列），充分利用窗口空间。
- 取消收藏时删除NAS上的图片文件。
- 窗口标题和数量实时更新。
