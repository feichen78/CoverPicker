🚨 核心开发规则（2026-07-23 更新）
提供完整代码：任何时候，需要提供修改好的完整代码（可以适当精简），以便用户直接覆盖。不允许仅给出片段代码、部分方法，让用户自行替换。输出完整代码时，不需要告诉用户"我只修改了xxx，但是为了完整，我还是提供xx"。如果代码没有变化，则不需要重新提供完整代码。

确认现象与需求：除非用户仅给出报错日志，其他时候必须向用户确认错误或问题的现象，以及用户的需求，用户同意后再开始修改代码。当用户提供了截图来说明问题时，如果读不懂一定要如实告知用户，并询问详细的错误，严禁自己胡乱解读、胡乱修改。当修改方案与用户需求完全一致时，不需要请用户确认。只有当方案与用户需求不同、或存在多种可选方案时，才需要向用户确认。严禁用户描述完现象，未经确认直接输出代码。

禁止擅自删改功能：任何时候，不允许未经用户授权，删除或修改coverpicker已有的功能、布局、按钮等。如果确实需要更改，必须取得用户确认。如果不确定用户原来的代码已实现什么功能，绝不允许擅自开始修改或生成简化的文件代码，务必向用户询问、索取原始代码。如有必要，可以增加调试日志，不需要请用户确认。每次调试，用户都会使用你最后提供的版本完整覆盖，不需要对此提出质疑，不需要就此向用户确认。

自查后输出：输出代码前必须进行三轮独立的审查：

第一轮（技术审查）：检查代码是否有语法错误、逻辑漏洞、变量名错误、缩进错误、导入缺失等技术性问题。

第二轮（需求审查）：以用户的视角模拟运行，在脑海中逐行执行代码，检查修改后的代码是否真正实现了用户描述的需求，而不是仅仅"能运行"。

第三轮（规则审查）：仔细核对前面3条规则，是否有违反任何一条。

三轮审查全部结束后，必须用中文输出一份简短的"需求实现确认清单"，至少包含，确认修改后的代码确实实现了用户的需求，且未违反核心规则。不允许输出"已自查3次，确认无误"这样空洞的结论，必须具体说明需求如何被实现。

1. 查看收藏窗口 - 初始显示窄线（问题5a）
现象：打开查看收藏窗口时，内容区域显示为一条窄线，图片不显示或闪现后消失。

根本原因：

FavoritesDialog.__init__ 中直接调用 _refresh_favorites()，此时窗口尚未显示，container_widget 宽度为 0。

QFlowLayout 在宽度为 0 时计算高度返回 0，导致滚动区域内容高度为 0。

FavImageLabel 未重写 sizeHint，导致布局无法获取正确尺寸。

修复方案：

将 _refresh_favorites() 延迟到 showEvent 中调用（QTimer.singleShot(50, ...)）。

改用 QGridLayout 替代 QFlowLayout，避免复杂的流式布局尺寸计算。

在 _refresh_favorites 末尾强制设置 container_widget.setFixedHeight()。

重写 FavImageLabel.sizeHint() 返回固定尺寸。

2. 查看收藏窗口 - 最大化右侧空白（问题5b）
现象：窗口最大化后，图片集中在左侧，右侧出现大量空白。

根本原因：

QFlowLayout 中硬编码 max_cols=5，且图片尺寸固定，导致无法充分利用窗口宽度。

图片尺寸未根据窗口宽度动态调整。

修复方案：

动态计算列数：cols = min(5, max_count)（最大收藏数量）。

图片宽度 = (可用宽度 - (列数-1)×间距) / 列数。

同时保证图片宽度 ≥ 5列满排时的宽度（(可用宽度 - 4×间距) / 5），避免单张图片过小。

使用 QGridLayout，列拉伸因子设为 0，保持左对齐。

3. 查看收藏窗口 - 绿点显示错误（问题5c）
现象：所有收藏截图都显示绿色圆点（已导出标记），但实际并未导出。

根本原因：

数据库 favorites 表中 is_exported 字段默认值为 0，但早期版本中插入收藏时可能错误地将 is_exported 设为 1。

旧数据污染导致所有收藏项都被标记为已导出。

修复方案：

在 FavoritesDialog.__init__ 中添加 _fix_old_exported() 方法。

遍历传入的收藏数据，将所有 exported=True 的记录重置为 False，并更新数据库。

同步更新 controller.favorites 中的对应项。

4. 查看收藏窗口 - 布局左对齐问题
现象：A区5张满行正常显示，B区4张和E区1张整体居中，而非左对齐。

根本原因：

QGridLayout 默认居中对齐，且列拉伸因子为 1 导致不均匀分布。

未显式设置布局和子 widget 的对齐方式。

修复方案：

设置 grid_layout.setAlignment(Qt.AlignLeft)。

设置 grid_layout.setColumnStretch(c, 0) 使列宽由 widget 决定。

添加 grid_layout.setAlignment(label, Qt.AlignLeft | Qt.AlignTop)。

间距 spacing_h = 2，接近主界面紧凑风格。

5. 收藏窗口细选 - 找不到对应截图位置
现象：在查看收藏窗口中选中一张图片，点击“细选”，提示“未找到对应截图位置”。

根本原因：

收藏项的时间点可能不在当前分区的截图列表中（因为收藏是从数据库恢复的，而截图列表是动态生成的）。

pos_in_segment 找不到时直接报错。

修复方案：

在 zoom_selected 中，如果 pos_in_segment 未找到，默认设为 0。

不再弹出警告，直接进入细选窗口（ZoomDialog 本身依赖 center_time 定位，pos 仅为辅助）。

6. 细选窗口 - 帧间隔不符合设计
现象：中心帧 3750.9s，前一张 3750.8s（相差0.1s），后一张 3752.6s（相差1.7s），分布不均匀。

根本原因：

zoom_dialog.py 中使用 random.uniform(start, end) 随机生成 9 个时间点，不是固定间隔。

与 PRODUCT.md 要求不符（应显示 -4s, -3s, -2s, -1s, 当前帧, +1s, +2s, +3s, +4s）。

修复方案：

新增 _get_time_offsets() 方法，根据层级返回固定偏移量：

L1: [-4, -3, -2, -1, 0, 1, 2, 3, 4]（间隔1秒）

L2: [-2, -1.5, -1, -0.5, 0, 0.5, 1, 1.5, 2]（间隔0.5秒）

L3: [-1, -0.75, -0.5, -0.25, 0, 0.25, 0.5, 0.75, 1]（间隔0.25秒）

L4: [-0.4, -0.3, -0.2, -0.1, 0, 0.1, 0.2, 0.3, 0.4]（间隔0.1秒）

7. 查看收藏窗口 - 多选细选无提示
现象：选中多张图片后点击“细选”，无任何提示，也无法进入细选。

根本原因：

_refresh_favorites 被 resizeEvent 频繁触发，在重建布局时清空了 selected_indices。

用户点击多张图片后，_refresh_favorites 被意外触发，导致选中状态丢失。

zoom_selected 被调用时 selected_indices 只剩 1 张，不满足多选条件，因此无提示。

修复方案：

在 _refresh_favorites 中保存当前选中项的标识 (seg_label, time)，重建后根据标识恢复选中状态。

不再简单清空 selected_indices，而是基于保存的标识重建。

8. 查看收藏窗口 - 导出目录记忆失效
现象：在查看收藏窗口中点击“导出”，默认目录是上次保存视频片段的目录，而非保存截图的目录。

根本原因：

使用同一个键 last_export_dir 存储图片和视频导出目录，导致互相覆盖。

修复方案：

新增独立键 last_image_export_dir 存储图片导出目录。

优先读取 last_image_export_dir，若不存在则回退到 last_export_dir。

9. 查看收藏窗口 - 移除替换原图按钮
需求：查看收藏窗口不需要“替换原图”按钮。

修复方案：

从 favorites_dialog.py 的底部操作栏中移除 replace_btn 相关代码。

移除 _update_button_states 中对 replace_btn 的状态更新。

10. 细选窗口 - 多选替换原图无提示
现象：在细选窗口中选中多张图片，点击“替换原图”，无任何提示。

根本原因：

_update_buttons 中 self.replace_btn.setEnabled(len(self.selected_indices) == 1)，多选时按钮被禁用，点击无效。

修复方案：

将 replace_btn 改为始终可用（setEnabled(True)）。

在 replace_selected 方法开头检查 len(self.selected_indices) != 1，弹出警告并返回。

与 zoom_btn 的行为保持一致。

11. 细选窗口 - 第一次进入异步冲突报错
现象：第一次进入细选窗口时，控制台报错 RuntimeError: Cannot enter into task ... while another task ... is being executed。

根本原因：

ZoomDialog.__init__ 中立即调用 self.load_candidates()，内部执行 asyncio.create_task(self._load_candidates_async())。

此时主事件循环中已有其他任务（如 SegmentView._load_video）正在执行，导致任务嵌套冲突。

修复方案：

将 self.load_candidates() 延迟执行：QTimer.singleShot(50, self.load_candidates)。

确保主事件循环先完成当前任务。

12. 自动备份 - 删除所有 .db 文件
现象：自动备份功能删除了所有备份文件，未保留任何备份。

根本原因：

delete_old_backups 只保留当日备份，若当日无备份则全部删除。

修复方案：

先找出所有备份文件，按修改时间降序排序。

保留最新的一个备份文件，删除所有非当日且非最新的文件。

确保至少保留一个最近的备份。

v2.5.0 查看收藏窗口与细选功能修复版

修复的问题
查看收藏窗口窄线问题：改用 QGridLayout，延迟刷新，固定容器高度。

查看收藏窗口右侧空白问题：动态列数计算，图片尺寸自适应。

查看收藏窗口绿点错误：_fix_old_exported() 修复旧数据。

查看收藏窗口布局左对齐：强制 AlignLeft，列拉伸为 0。

收藏窗口细选位置找不到：pos_in_segment 找不到时默认 0。

细选窗口帧间隔不符合设计：固定偏移量，L1~L4 按设计间隔。

查看收藏窗口多选细选无提示：_refresh_favorites 保留选中状态。

查看收藏窗口导出目录记忆失效：独立键 last_image_export_dir。

查看收藏窗口移除替换原图按钮：已移除。

细选窗口多选替换原图无提示：按钮始终可用，方法内检查。

细选窗口第一次进入异步冲突：QTimer.singleShot 延迟加载。

自动备份删除所有 .db 文件：至少保留一个最新备份。

关键调试经验
蓝点与 selected_indices 的一致性：蓝点准确反映选中状态，问题出在 _refresh_favorites 意外清空 selected_indices。

按钮禁用 vs 方法内检查：按钮应始终可用，在方法内检查条件并给出提示，而非禁用按钮。

异步任务延迟执行：__init__ 中立即创建异步任务可能导致冲突，应延迟执行。

布局重建时保留状态：重建 UI 时应先保存关键状态，重建后恢复。
